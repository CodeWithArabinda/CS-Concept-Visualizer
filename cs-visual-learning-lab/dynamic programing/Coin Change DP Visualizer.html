Algorithm Theory

Building Combinations

Coin change is a classic DP problem. We build the solution up iteratively by analyzing every amount from 0 up to our target for each coin available.

By iterating through coins first, we guarantee we only count combinations (e.g., [1,2]) and not permutations (e.g., [1,2] and [2,1]).

Two Distinct Problems

  • 1. Total Number of Ways Finds how many distinct ways we can make the target amount.
    dp[i] += dp[i - coin]
  • 2. Minimum Coins Required Finds the smallest handful of coins that sum exactly to the target.
    dp[i] = min(dp[i], dp[i-coin] + 1)

Complexity Overview

Time Complexity O(C × A)
Space Complexity O(A)

Where C is the number of coins and A is the target amount.
