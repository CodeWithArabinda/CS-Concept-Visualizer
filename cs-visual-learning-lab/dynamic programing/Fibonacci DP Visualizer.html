Algorithm Theory

Overlapping Subproblems

In a naive recursive approach, computing F(4) requires computing F(3) and F(2). However, F(3) also requires computing F(2). This results in calculating the same values repeatedly.

Dynamic Programming

DP avoids this by storing the result of each subproblem the first time it's computed. The next time it's needed, it simply looks up the value in $O(1)$ time.

Tabulation (Bottom-Up): We build a table (array) iteratively from the smallest subproblems F[0] and F[1] up to F[n].

Complexity Comparison

Naive Recursion Time: O(2&spadesuits;ⁿ) Exponential growth. Extremely slow for N > 40.
DP Time Complexity: O(n) Linear growth. Instantly calculates massive N.
